## ✅ Реализовано

### Внешняя инфраструктура

- [x] Куплен домен `nexus-platform.ru`
- [x] Зарегистрирован Telegram-канал `@NexusPlatformRu`

### Инфраструктура

- [x] Микросервисная архитектура (Python + FastAPI)
- [x] Единый базовый класс `BaseAgent` с поддержкой tool calling и автоматическим добавлением описания инструментов в system prompt
- [x] API Gateway (порт 8000) для единого входа
- [x] Process Manager (порт 8008) для запуска/остановки агентов и отслеживания статуса
- [x] Конфигурация агентов через YAML (имя, системный промпт, инструменты, порт и т.д.)

### Агенты

- [x] **Михаил** (секретарь) — заметки, список покупок, напоминания, задачи (с интеграцией в таск-трекер)
- [x] **Елена** (семейный советник) — данные о семье, подарки, события
- [x] **Виктор** (архитектор) — реестр агентов, создание структуры новых агентов (базовый)
- [x] **Ментор** — помощь в разработке (базовый)
- [x] **Дизайнер** — генерация UI-компонентов (интегрирован с Vue)
- [x] **Учитель английского** — обучение, словарь, прогресс (базовый)

### Таск-трекер

- [x] Микросервис `task-manager` (порт 8009) с SQLite, CRUD для задач
- [x] Модель задачи: id, title, description, status, priority, assignee, project, timestamps
- [x] Канбан-доска в Nexus (страница `/tasks`) с колонками «Новые», «В работе», «Завершённые»
- [x] Интеграция с Михаилом: инструменты `create_task`, `get_tasks`
- [x] Фильтрация задач по проекту (параметр `project` в `get_tasks`)

### Nexus (фронтенд)

- [x] Vue 3 + Tailwind CSS, Pinia (заложена)
- [x] Страницы: дашборд, агенты, чаты, логи, задачи, статистика, биллинг, профиль, библиотека
- [x] Дашборд с карточками статистики, последними логами, активными задачами, состоянием агентов
- [x] Страница агентов (`/agents`) — карточки с кнопками включения/выключения, перезагрузки, отображением порта, PID, uptime
- [x] Состояние агента определяется по реальным запущенным процессам (не из стора)
- [x] Массовые кнопки «Запустить всех» / «Остановить всех»
- [x] Ручное и автоматическое обновление списка агентов
- [x] Страница логов (`/logs`) — выбор агента, загрузка логов, поиск по содержимому
- [x] Мониторинг gateway и process-manager на дашборде
- [x] Чат (`/chats`) с выбором агента, отправкой сообщений, отображением кода, копированием
- [x] Загрузка истории сообщений при выборе чата (через AgentsService)
- [x] Стилизация, кастомные цвета, иконки-эмодзи, спиннеры загрузки

### Общее

- [x] Единое логирование агентов в `~/ai-agents/logs/<agent_id>/agent.log` с ротацией
- [x] Единое логирование для сервисов **gateway, process-manager, task-manager** в `~/ai-agents/logs/<service>/service.log` с ротацией
- [x] Сборка истории диалогов в `conversations.json` для каждого агента
- [x] Скрипты `run-nexus.sh` и `stop-nexus.sh` для одновременного запуска/остановки всех компонентов

## 🌟 Среднесрочные идеи

- [ ] **Мета-агент Nexus Core** — центральный ИИ, который ассоциирует себя с системой, управляет всеми агентами, хранит единый контекст и выступает «точкой притяжения»
- [ ] **Инвестиционный советник** — сбор данных, анализ, рекомендации
- [ ] **Планировщик времени** — интеграция с календарём, умное расписание
- [ ] **Библиотекарь** — учёт книг/аудиокниг, рекомендации
- [ ] **Копирайтер** — генерация текстов, постов, статей
- [ ] **Автора песен** — генерация песен по жанрам
- [ ] **Повар-диетолог** — составление плана питания, меню на неделю, расчёт калорийности, поиск и подготовка рецептов
- [ ] Векторная память для всех агентов (ChromaDB) с семантическим поиском по диалогам
- [ ] Мобильное приложение на Flutter (клиент для Nexus)
- [ ] Голосовой интерфейс (распознавание речи + синтез)
- [ ] Интеграция с внешними сервисами (Google Calendar, Notion, Telegram-каналы и т.д.)
- [ ] Аутентификация и мультипользовательность
- [ ] Улучшение инфраструктуры запуска — Docker

- [ ] Анимация для логотипа Nexus
- [ ] Визуальный конструктор агентов (drag-and-drop интерфейс для создания без кода)
- [ ] Интеграция дизайн-агента с Figma
- [ ] Для чатов и логов использовать WebSocket (вместо периодического опроса)

## 🐞 Известные проблемы

- [ ] Иногда модель не вызывает инструменты без `force_tool` (требуется доработка промпта или логики вызова)
- [ ] В логах агентов могут появляться дублирующиеся записи из-за нескольких хендлеров
- [ ] При длинных диалогах модель может «забывать» вызывать инструменты (проблема контекста)
- [ ] Страница «Чаты» — вёрстка уезжает при длинном диалоге (список слева должен оставаться на месте)
